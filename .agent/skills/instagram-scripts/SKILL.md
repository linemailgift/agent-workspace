# Instagram台本生成スキル（instagram-scripts）

## 概要
クライアントのYouTube動画の文字起こし・既存コンテンツ・強み分析をもとに、Instagram Reels用の1分台本を30本前後生成し、構造マップつきのHTMLページとして納品する。

## 使い方
ユーザーが「Instagramの台本を作って」「リールの原稿を作りたい」等と依頼した時に発動。
クライアント名と素材（文字起こし・議事録・提案書など）のパスを確認して開始する。

---

## 実行手順

### Step 1: 素材の収集・分析
クライアントのフォルダから以下を読み込む:

1. **YouTube文字起こし** — 話し方、口癖、独自フレーズ、比喩表現を抽出
2. **既存の戦略提案・競合分析** — ポジショニング、強み、ターゲット層を把握
3. **議事録・事業概要** — メソッド名、実績数字、エピソードを収集

#### 抽出すべき要素:
- **口癖・定型フレーズ**（冒頭の挨拶、締めの言葉、キャッチフレーズ）
- **具体的な数字**（指導人数、経験年数、受講生の成果など）
- **独自メソッド名**（○○ステップ、○○法など）
- **比喩表現**（RPG、スポーツ、日常生活の例えなど）
- **実話エピソード**（受講生ストーリー、自身の体験談）
- **CTA素材**（LINE特典、無料プレゼント、キーワードなど）

### Step 2: コンテンツカテゴリの設計
分析結果をもとに6つのカテゴリに台本を分類する:

| カテゴリ | 目的 | 本数目安 |
|---------|------|---------|
| A. 衝撃フック系 | スクロール停止 → 認知獲得 | 5-7本 |
| B. メソッド紹介系 | 専門性の証明 → 「もっと知りたい」 | 5-7本 |
| C. ストーリー系 | 感情を動かす → 共感・フォロー | 5-7本 |
| D. マインド系 | 考え方の変化 → 長期フォロワー化 | 4-5本 |
| E. ターゲット別（保護者など） | 別ペルソナへの訴求 → 高転換率 | 3-5本 |
| F. クイズ/あるある系 | エンゲージメント獲得 → 拡散 | 3-4本 |

**カテゴリはクライアントの事業に合わせて調整する。**
例: コーチング系ならストーリー多め、物販系ならビフォーアフター系を追加。

### Step 3: 台本の作成
各台本は以下の5ブロック構成で1分に収まるように書く:

```
【0:00-0:03】フック（3秒）
→ スクロールを止める衝撃の1文。疑問文 or 断言。

【0:03-0:10】問題提起（7秒）
→ 共感 or 恐怖で「自分ごと化」させる。

【0:10-0:45】本題（35秒）
→ 1つだけ伝える。具体例つき。クライアントの口癖・比喩を活用。

【0:45-0:55】まとめ（10秒）
→ 結論を1文で繰り返す。

【0:55-1:00】CTA（5秒）
→ 「保存して」「フォローして」「コメントで教えて」「プロフィールのリンクから」
```

#### 台本作成のルール:
- **クライアントの実際の話し方に寄せる**（文字起こしから抽出した口癖・フレーズを使う）
- **1本1メッセージ**（複数のことを詰め込まない）
- **具体的な数字を入れる**（「たくさん」ではなく「1000人」「14年」「2万枚」）
- **比喩は本人が使うものを優先**（勝手に新しい比喩を作らない）
- **CTAは3種類をローテーション**: 保存誘導 / フォロー誘導 / コメント誘導
- **LINE特典やプロフィールリンクへの誘導は自然に**（押し売り感を出さない）

### Step 4: HTMLマップの作成

#### ファイル構成:
- メインファイル: `クライアント/[名前]/Instagram台本マップ.html`

#### デザイン: Instagram風
```css
/* カラー */
背景: #fafafa（Instagramの標準背景）
テキスト: #262626
サブテキスト: #8e8e8e
ボーダー: #dbdbdb
リンク: #0095f6（Instagram Blue）
ヘッダーグラデーション: linear-gradient(135deg, #833ab4, #fd1d1d, #fcb045)

/* UI */
角丸: 16px（カード）、24px（ボタン）、12px（タグ）
影: なし or 極薄（box-shadow: 0 4px 20px rgba(0,0,0,0.08) on hover）
フォント: -apple-system, 'Hiragino Sans', 'Noto Sans JP'
ツールバー: sticky + backdrop-filter: blur(12px)
```

#### タグのカラー:
```
衝撃フック: background: #fce4ec, color: #c62828
メソッド:   background: #e8f5e9, color: #2e7d32
ストーリー: background: #e3f2fd, color: #1565c0
マインド:   background: #fff8e1, color: #f57f17
保護者向け: background: #f3e5f5, color: #7b1fa2
クイズ:     background: #e0f7fa, color: #00838f
```

#### HTMLの構成要素:

1. **ヘッダー** — Instagramグラデーション枠 + 白背景 + IGアイコン + タイトル
2. **フィルターバー**（sticky） — カテゴリ別絞り込みボタン + 表示件数
3. **コンテンツ構造マップ** — 6カテゴリのカード。各カードに台本リスト。クリックで該当台本へスクロール
4. **台本テンプレート解説** — 5ブロック構成の図 + 編集のコツ
5. **台本一覧（メイン）** — カードグリッド。各カードに5ブロックの台本。**全ブロック contenteditable="true"**
6. **フッター**

#### 機能:
- **contenteditable** — 台本テキストをブラウザ上で直接編集可能にする
- **filterScripts(cat)** — JavaScriptでカテゴリフィルタリング
- **scrollToCard(id)** — 構造マップのリストクリックで台本へスムーススクロール

### Step 5: GitHub Pagesにデプロイ

```bash
# docsフォルダにコピー（gitignore回避）
mkdir -p docs/[英語名]
cp "クライアント/[名前]/Instagram台本マップ.html" "docs/[英語名]/instagram-scripts.html"
git add "docs/[英語名]/instagram-scripts.html"
git commit -m "[クライアント名]Instagram台本マップ: 1分リール×[本数]本"
git push
```

**公開URL:** `https://linemailgift.github.io/agent-workspace/docs/[英語名]/instagram-scripts.html`

### Step 6: 完了処理
1. GitHub Pagesの反映を確認（60秒待ってWebFetchで確認）
2. 公開URLを表示
3. 完了時刻を表示
4. 通知音: `afplay /System/Library/Sounds/Glass.aiff`

## 台本の品質チェックリスト

台本作成後、以下を確認:

- [ ] 各台本が1分（約250-300文字）に収まっているか
- [ ] フックの最初の1文が「スクロールを止める力」を持っているか
- [ ] 1本1メッセージになっているか（詰め込みすぎていないか）
- [ ] クライアント本人の口癖・比喩が使われているか
- [ ] 具体的な数字が入っているか
- [ ] CTAが3種類（保存/フォロー/コメント）でローテーションしているか
- [ ] 全カテゴリに偏りなく台本があるか
- [ ] contenteditable が全ブロックに付いているか
- [ ] フィルター機能が正しく動作するか

---

## 応用: 既存台本からの派生

台本30本を作った後、追加で以下の提案も可能:

- **カルーセル投稿用**: 1本の台本を5-10枚のスライドに分解
- **ストーリーズ用**: 台本のフック部分だけを15秒に切り出し
- **投稿スケジュール**: 30本を4-6週間の投稿カレンダーに配置
