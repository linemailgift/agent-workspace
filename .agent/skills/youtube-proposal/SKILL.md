# YouTube提案一式スキル（youtube-proposal）

## 概要
クライアントのYouTubeチャンネルを分析 → 戦略レポート・特典企画書・Instagram台本30本の提案一式を作成し、GitHub Pagesで公開する。

## 使い方
ユーザーが「YouTube分析して提案一式作って」「YouTubeから戦略レポート作りたい」等と依頼した時に発動。
クライアント名とYouTubeチャンネルURLを確認して開始する。

---

## インプット

**必須:**
1. クライアント名
2. YouTubeチャンネルURL

**推奨（精度UP）:**
3. 文字起こしテキスト 5〜10本分
4. 打合せメモ・議事録

**あればベター:**
5. Instagram URL / HP URL / 競合チャンネルURL

---

## 全体フロー

```
Step 0: インプット収集（何が足りないか確認）
  ↓
Step 1: YouTube分析（チャンネル・動画・話し方・ターゲット）
  ↓  → YouTube分析.md（内部用）
Step 2: ペルソナ定義（発信者像・トーン・口癖・価値観）
  ↓  → ペルソナ定義.md（内部用）
Step 3: YouTube戦略レポート（HTML）
  ↓  → youtube-report.html ★クライアント提示用
Step 4: 特典コンテンツ企画書（HTML）
  ↓  → bonus-plan.html ★クライアント提示用
Step 5: Instagram台本30本マップ（HTML）
  ↓  → instagram-scripts.html ★クライアント提示用
Step 6: GitHub Pagesデプロイ + URL発行
```

---

## フェーズ分割（通しでも分割でも実行可能）

| フェーズ | ステップ | 成果物 |
|---------|---------|--------|
| Phase A: 分析 | Step 0-2 | YouTube分析.md + ペルソナ定義.md |
| Phase B: レポート | Step 3-4 | youtube-report.html + bonus-plan.html |
| Phase C: 台本 | Step 5-6 | instagram-scripts.html + デプロイ |

Phase Aの成果物がPhase B-Cのインプットになる。

## 実行手順

### Step 0: インプット収集

ユーザーに以下を確認する:

1. **クライアント名**（日本語 + 英語名）
2. **YouTubeチャンネルURL** → WebFetchでチャンネル情報を取得
3. **文字起こしはあるか？** → あればパスを聞く。なければYouTube動画の情報から分析
4. **議事録・打合せメモはあるか？** → あればパスを聞く
5. **Instagram / HP / 競合URL** → あれば聞く

足りない情報があっても最低限（名前 + YouTubeURL）で開始可能。

### Step 1: YouTube分析

YouTubeチャンネルと動画を分析し、以下を整理する。

#### 分析項目:

**チャンネル概要:**
- チャンネル名・登録者数・総動画数
- チャンネル説明文
- メインジャンル・サブジャンル

**動画分析（上位10-20本）:**
- タイトルのパターン（疑問形？ 数字入り？ 煽り系？）
- 再生数の多い動画の共通点
- サムネイルの傾向（テキスト多め？ 顔出し？ ビフォーアフター？）
- 動画の長さの傾向

**話し方の特徴（文字起こしがある場合）:**
- 口癖・定型フレーズ
- テンション（落ち着いた系 / 元気系 / 淡々系）
- 比喩表現の傾向
- 冒頭の入り方パターン
- CTAの言い方

**ターゲット分析:**
- 誰に向けて発信しているか
- コメント欄から推測される視聴者層
- 届いている層 vs 届けるべき層のギャップ

#### 出力:
`クライアント/[名前]/YouTube分析.md` に保存。

```markdown
# [クライアント名] YouTube分析

## チャンネル概要
- チャンネル名:
- 登録者数:
- 総動画数:
- メインジャンル:

## 動画傾向
### 再生数TOP5
1. [タイトル] — [再生数]
...

### タイトルパターン
- ...

### サムネ傾向
- ...

## 話し方の特徴
### 口癖・定型フレーズ
- ...

### テンション・トーン
- ...

### 比喩表現
- ...

## ターゲット分析
### 現在のターゲット
- ...

### 理想のターゲット
- ...

## 強み・差別化ポイント
- ...

## 課題・改善ポイント
- ...
```

### Step 2: ペルソナ定義

Step 1の分析結果 + 文字起こし + 議事録から、クライアントのペルソナを定義する。

#### 定義する項目:

- **基本情報:** 名前、肩書、専門分野、事業内容
- **ポジショニング:** ニッチ、差別化ポイント、競合との違い
- **発信トーン:** テンション、語尾の癖、敬語/タメ口のバランス
- **口癖・キーフレーズ:** 冒頭の挨拶、締めの言葉、決め台詞
- **価値観・信念:** 何を大切にしているか、NGなこと
- **具体的な数字:** 経験年数、指導人数、実績数字
- **独自メソッド:** メソッド名、ステップ名、フレームワーク
- **比喩・例え:** よく使う比喩表現
- **実話エピソード:** 受講生の変化、自身の体験談
- **CTA素材:** LINE特典、無料プレゼント、キーワード

#### 出力:
`クライアント/[名前]/ペルソナ定義.md` に保存。

yoko-nail-personaスキルの構造を参考にする。このペルソナ定義は後続のStep 3-5で一貫したトーンを保つために使う。

### Step 3: YouTube戦略レポート（HTML）

分析結果をクライアント提示用のHTMLレポートにまとめる。

#### レポート構成:

1. **表紙** — クライアント名 + 「YouTube戦略レポート」+ 日付
2. **チャンネル概要** — 登録者・動画数・平均再生数をカード表示
3. **強み分析** — 話し方の特徴・差別化ポイントを図解
4. **ターゲット分析** — 誰に届いているか・届けるべきかを比較
5. **コンテンツ傾向** — 伸びるテーマ・反応パターンをグラフ風に
6. **改善提案** — タイトル・サムネ・概要欄・投稿頻度の具体的提案
7. **特典企画案** — LINE登録特典3案のサマリー（詳細はStep 4）
8. **SNS展開戦略** — YouTube→Instagram→LINE→個別相談の導線図

#### デザイン:
slide-presentationスキルのデザインシステムを流用する。

```
カラー: 紫グラデーション（#8B5CF6 → #7C3AED）
フォント: Noto Sans JP
レイアウト: スライド形式（矢印キーで切替）
コンポーネント: カプセル見出し、円形アイコン、左縦バー見出し、薄紫パネル
ナビゲーション: ◀ ▶ ボタン + ページ番号
アニメーション: fadeInUp
フッター: produced by いちじく
```

#### 出力:
`pages/[英語名]/youtube-report.html` に保存。

### Step 4: 特典コンテンツ企画書（HTML）

YouTube分析から導き出した特典（LINE登録特典等）を3案提案する。

#### 企画書の構成:

1. **表紙** — 「特典コンテンツ企画書」+ クライアント名 + 日付
2. **なぜ特典が必要か** — YouTube→LINE→個別相談の導線における特典の役割
3. **特典3案** — カード形式で並べる

各案に含める情報:
- **タイトル**（キャッチーな特典名）
- **形式**（PDF / 動画 / 診断 / チェックリスト / テンプレート等）
- **ボリューム**（ページ数・分数の目安）
- **構成**（章立て or ステップ）
- **刺さる理由**（YouTube分析の根拠を明記）
- **制作難易度**（★☆☆ / ★★☆ / ★★★）
- **制作期間の目安**

4. **おすすめ案** — 3案のうち最も効果的な案を推薦 + 理由
5. **次のステップ** — 特典制作→LINE構築→YouTube導線設計の流れ

#### デザイン:
Step 3と同じデザインシステム（slide-presentation）を使用。

#### 出力:
`pages/[英語名]/bonus-plan.html` に保存。

### Step 5: Instagram台本30本マップ（HTML）

instagram-scriptsスキルの手順に従って台本30本を作成する。

#### 重要:
- Step 1のYouTube分析とStep 2のペルソナ定義をインプットとして使う
- instagram-scriptsスキルのStep 2〜5の手順をそのまま実行する
- カテゴリ設計はクライアントの事業に合わせて調整

#### 出力:
`pages/[英語名]/instagram-scripts.html` に保存。

### Step 6: GitHub Pagesデプロイ + URL発行

3つのHTMLファイルをまとめてデプロイする。

```bash
cd "リポジトリルート"
git add pages/[英語名]/youtube-report.html
git add pages/[英語名]/bonus-plan.html
git add pages/[英語名]/instagram-scripts.html
git commit -m "[クライアント名]YouTube提案一式: 戦略レポート・特典企画・Instagram台本30本"
git push origin main
```

#### 公開URL（3つまとめて表示）:
```
戦略レポート: https://linemailgift.github.io/agent-workspace/pages/[英語名]/youtube-report.html
特典企画書:   https://linemailgift.github.io/agent-workspace/pages/[英語名]/bonus-plan.html
Instagram台本: https://linemailgift.github.io/agent-workspace/pages/[英語名]/instagram-scripts.html
```

#### 完了処理:
1. GitHub Pagesの反映を確認（60秒待ってWebFetchで確認）
2. 3つの公開URLをまとめて表示
3. openコマンドでブラウザを開く
4. 完了時刻を表示
5. 通知音: `afplay /System/Library/Sounds/Glass.aiff`

## 品質チェックリスト

### Phase A（分析）完了時:
- [ ] YouTube分析.mdにチャンネル概要・動画傾向・話し方・ターゲットが網羅されているか
- [ ] ペルソナ定義.mdに口癖・トーン・価値観・数字・メソッドが入っているか
- [ ] 2ファイルの整合性（矛盾がないか）

### Phase B（レポート）完了時:
- [ ] 戦略レポートの提案がYouTube分析の根拠に基づいているか
- [ ] 特典3案がターゲット分析と整合しているか
- [ ] スライドのデザインがslide-presentationと統一されているか
- [ ] ナビゲーション（矢印キー）が動作するか

### Phase C（台本）完了時:
- [ ] 台本がペルソナ定義の口癖・トーンを反映しているか
- [ ] instagram-scriptsスキルの品質チェックリストをすべてクリアしているか
- [ ] 3つのHTMLすべてがGitHub Pagesで正しく表示されるか
